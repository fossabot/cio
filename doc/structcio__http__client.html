Detailed Description

A cio_http_client struct represents an HTTP client connection.

A cio_http_client is automatically created by the cio_http_server after a successfully established TCP connection. The memory allocation is done by the alloc_client function provided during initialization of a cio_http_server. The resources allocated for the HTTP client are automatically freed when the client connection is closed by using the free_client function provided during initialization of a cio_http_server.

struct cio_write_buffer cio_http_client::wbh

A write buffer head to write data to an HTTP client.

Typically you will not need direct access to the write buffer, use write_header and write_response to send "normal" HTTP responses to a client.

If you have an upgraded HTTP connection (i.e. web sockets), you are encouraged to use this write buffer.

uint64_t cio_http_client::content_length

The content length of the client request.

Reading the content length is only possible after the corresponding header fields were read by the HTTP parser. After you got the on_headers_complete callback you can rely on the field being set.
