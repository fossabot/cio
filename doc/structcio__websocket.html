enum cio_error(* cio_websocket::close)(struct cio_websocket *ws, enum cio_websocket_status_code status, struct cio_write_buffer *reason)

Closes a websocket.

Warning
Please note that the data reason encapsulates will be scrambled by the library if this function is used in a websocket client connection. You should ALWAYS intialize the write buffer elements in * reason using the non-const initialization function.
Parameters
wsThe websocket to be closed.
statusThe websocket status code to be sent.
reasonA buffer which contains the reason for the close in an UTF8 encoded string. Could be NULL if no reason should be sent.
Returns
CIO_SUCCESS for success, CIO_MESSAGE_TOO_LONG if length of reason exceeds a small websocket frame.

A pointer to a function which is called if a ping frame was received.

Library users are note required to set this function pointer. If you set this function pointer, please do NOT sent a pong frame as a response. The library takes care of this already.

Parameters
wsThe websocket which received the ping frame.
dataThe data the ping frame carried.
lengthThe length of data the ping frame carried.

void(* cio_websocket::on_close)(const struct cio_websocket *ws, enum cio_websocket_status_code status, const char *reason, size_t reason_length)

A pointer to a function which is called when a close frame was received.

Library users are note required to set this function pointer. If you set this function pointer, please do NOT close() the websocket in the callback function. This is done immediately by the library after return of this function.

Parameters
wsThe websocket which received the pong frame.
dataThe data the pong frame carried.
lengthThe length of data the pong frame carried.
void(* cio_websocket::on_error)(const struct cio_websocket *ws, enum cio_websocket_status_code status, const char *reason)

A pointer to a function which is called if a receive error occurred.

Library users are note required to set this function pointer. If you set this function pointer, please operate no longer on this websocket (Do not call close() etc. on the websocket). Immediately after this function returns, the library closes the websocket on its own.

Parameters
wsThe websocket which encountered the error.
statusThe status code describing the error.
statusA string describing the reason for the error. Could be NULL.
void(* cio_websocket::write_textframe)(struct cio_websocket *ws, struct cio_write_buffer *payload, bool last_frame, cio_websocket_write_handler handler, void *handler_context)

Writes a text frame to the websocket.

Warning
Please note that the data payload encapsulates will be scrambled by the library if this function is used in a websocket client connection. So if you want to write the same data again, you have to re-initialize the data encapsluated by payload. In addition you should ALWAYS intialize the write buffer elements in payload using the non-const initialization function.
If you call this function consecutively without waiting that handler is called, you should know that handler is called only for the last write call!
Parameters
payloadThe payload to be sent.
last_frametrue if the is an unfragmented message or the last frame of a fragmented message, false otherwise.
handlerA callback function that will be called when the write completes.
handler_contextA context pointer given to handler when called.
