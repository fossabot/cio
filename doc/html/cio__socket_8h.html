Detailed Description

This file contains the interface of a socket.

You can get an I/O stream from a socket, close the socket or set several socket options.

CIO_EXPORT enum cio_error cio_socket_init ( struct cio_socket *  socket,
enum cio_address_family  address_family,

uint64_t  close_timeout_ns,
cio_socket_close_hook  close_hook 
)

Initializes a cio_socket.

Warning
Please note that close_hook will NOT be called if this functions fails.
Parameters
socketThe cio_socket that should be initialized.
address_familyThe address family like CIO_ADDRESS_FAMILY_INET4 or CIO_ADDRESS_FAMILY_INET6.
loopThe event loop the socket shall operate on.
close_timeout_nsThe timeout in ns until a closed TCP connection waits for a TCP FIN packet from the remote peer before sending a TCP RST.If you set this parameter to >0, you can effectivly control how long the TCP socket stays in the FIN_WAIT_2 state. Setting this parameter to 0 leaves it up to the operating system to get the socket out of FIN_WAIT_2.
close_hookA close hook function. If this parameter is non NULL, the function will be called directly after closing the cio_socket. It is guaranteed the the cio library will not access any memory of cio_server_socket that is passed to the close hook. Therefore the hook could be used to free the memory of the server socket.
Returns
CIO_SUCCESS for success.

Parameters

onWhether or not to enable TCP keepalives.
keep_idle_sTime in seconds the connections needs to remain idle before start sending keepalive messages. This option might be unused in some platform implementations.
keep_intvl_sTime in seconds between individual keepalive probes. This option might be unused in some platform implementations.
