Detailed Description

Generic interface to write user specific HTTP handlers.

You have to set the callback functions with respect to the HTTP data (header information and/or body) you are interested in. If you don't set any callback, the HTTP server will return "500 Internal server error" to the requesting client.

Inside the callback functions you have access to the HTTP client which performed the request. You have to use the client functions write_response and write_header to write HTTP responses (including a body) or just a simple response header, respectively.

In case of an upgraded HTTP connection, callback have access to the buffered stream used by the HTTP client and the companion read buffer. This provides a convenient way to take over a complete HTTP connection. Please note that for upgraded connections this handler is responsible to close the client connection!

Enumeration Type Documentation

enum cio_http_cb_return

The cio_http_cb_return enum lists the allowed return values of user specified callback functions like on_header_field. If the data provided via the callbacks doesn't match to the protocol the HTTP handler handles, return cio_http_cb_error, cio_http_cb_success otherwise.

Enumerator
